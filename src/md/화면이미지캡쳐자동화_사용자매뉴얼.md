# 매뉴얼 이미지 자동화 도구 - 사용자 가이드

이 가이드는 `scripts/capture_manual_images.js` 도구를 사용하여 Stitch 프로젝트의 매뉴얼용 스크린샷을 자동으로 생성하는 방법을 설명합니다.

## 1. 사전 준비 사항 (Prerequisites)

### 1.1 Node.js 환경
- 컴퓨터에 Node.js가 설치되어 있어야 합니다.

### 1.2 프로젝트 의존성 설치
터미널에서 프로젝트 루트 경로(`c:\AI_DEV\stitch`)로 이동한 후 다음 명령어를 실행하여 필요한 라이브러리(Puppeteer 등)를 설치합니다.
```bash
npm install
```

### 1.3 로컬 개발 서버 실행
스크린샷을 찍을 대상인 웹 애플리케이션이 실행 중이어야 합니다. **새로운 터미널 창**을 열고 다음을 실행하세요.
```bash
npm run dev:all
```
* 서버가 `http://localhost:5173`에서 정상적으로 실행 중인지 확인하세요.

---

## 2. 도구 실행 방법 (How to Run)

### 2.1 스크립트 실행
터미널에서 다음 명령어를 입력하여 자동화 도구를 시작합니다.
```bash
node scripts/capture_manual_images.js
```

### 2.2 로그인 수행 (필요한 경우)
1. 1. 스크립트가 실행되면 크롬 브라우저 창이 자동으로 열립니다.
2. **"로그인 상태 식별 중..."** 메시지가 뜨고 잠시 대기합니다.
3. 만약 **로그인이 필요한 상태**라면, 터미널에 다음과 같은 메시지가 나타납니다:
   > 🔒 [로그인 필요] 로그인 화면이 감지되었습니다.
   > 👉 브라우저에서 직접 로그인을 진행해주세요!
4. 열린 브라우저 창에서 **직접 이메일과 비밀번호를 입력하여 로그인**하세요.
5. 로그인이 완료되고 대시보드 화면으로 넘어가면, 스크립트가 이를 자동으로 감지하고 촬영을 시작합니다.

### 2.3 캡처 진행 확인
- 스크립트가 자동으로 페이지를 이동하며 버튼을 클릭하고 스크린샷을 찍습니다.
- 터미널 로그를 통해 진행 상황을 확인할 수 있습니다:
  > 📸 촬영 중: 투자관리_조회...
  > └─ 저장됨: 01_investment_list_mockup.png

### 2.4 결과 확인
- 모든 작업이 완료되면 `🎉 모든 캡처가 완료되었습니다!` 메시지가 뜹니다.
- 생성된 이미지는 다음 경로에서 확인할 수 있습니다:
  - `src/md/images/`

## 3. 문제 해결 (Troubleshooting)

### Q1. "버튼 클릭 실패: 요소를 찾을 수 없습니다" 경고가 떠요.
- **원인**: UI가 변경되어 버튼의 텍스트가 바뀌었거나, 화면 로딩이 느려 버튼이 아직 뜨지 않은 경우입니다.
- **해결**:
  - `src/LanguageContext.jsx` 파일에서 해당 버튼의 한글 텍스트(예: '투자내역조회')가 정확한지 확인하세요.
  - `scripts/capture_manual_images.js` 파일의 `SCENARIOS` 부분에서 버튼 텍스트를 수정하세요.

### Q2. 로그인 후에도 "로그인 필요" 상태에서 넘어가지 않아요.
- **원인**: 네트워크가 느리거나 네비게이션(Nav) 요소가 늦게 로딩되는 경우일 수 있습니다.
- **해결**: 브라우저를 새로고침하거나, 스크립트를 `Ctrl+C`로 종료하고 다시 실행해보세요.

### Q3. 브라우저가 열리지 않아요.
- **원인**: Puppeteer 설치 문제일 수 있습니다. `node_modules` 폴더를 삭제하고 `npm install`을 다시 수행해보세요.
