# 제안서: 탭(Tab) 기반 내비게이션 레이아웃 구현

## 1. 목표 (Goal)
사용자는 웹 브라우저나 IDE(개발툴)처럼 동작하는 **"탭 인터페이스"**를 원합니다.
- 사이드바 메뉴 클릭 -> 상단에 **탭(Tab)** 추가 및 열기.
- 열린 탭들은 상단에 유지되어 빠르게 전환 가능.
- 사이드바는 항상 화면 왼쪽에 고정.

## 2. 현재 상태 (Current Status)
- **대시보드**: 사이드바를 자체적으로 가지고 있음.
- **상점/랭킹/내 몬스터**: 각각 독립된 페이지로, "뒤로 가기" 버튼이 있음.
- **이동 방식**: 페이지 전체가 깜빡이며 이동하는 전통적인 방식.

## 3. 변경 제안 (Refactoring Plan)

### 1단계: `MainLayout.jsx` (새로운 틀 만들기)
앱의 뼈대가 될 새로운 레이아웃 컴포넌트를 만듭니다:
1.  **사이드바**: 왼쪽에 고정 (항상 보임).
2.  **탭 바 (Tab Bar)**: 메인 화면 최상단에 위치. 열린 페이지 목록 표시.
3.  **컨텐츠 영역**: 탭 아래 공간. 현재 선택된 탭의 내용물 표시.

### 2단계: 탭 로직 (State Management)
- 관리해야 할 상태: `열린 탭 목록(openTabs)`, `현재 활성 탭(activeTab)`.
- **동작 방식**:
    - 사이드바 클릭 -> 탭 목록에 없으면 추가, 있으면 해당 탭으로 이동.
    - 탭 클릭 -> 화면 전환.
    - 'X' 버튼 클릭 -> 탭 닫기 (닫으면 옆에 있는 탭으로 이동).

### 3단계: 페이지 리팩토링 (다이어트)
- **Dashboard.jsx**: 내부에 있던 사이드바 코드를 삭제. (이제 공통 레이아웃이 담당하므로)
- **Shop/Leaderboard/MyMonster**: 상단의 "뒤로 가기" 헤더 삭제. (탭으로 이동하므로 불필요)

### 4단계: 라우팅 구조 변경 (`App.jsx`)
`MainLayout` 안에 모든 페이지를 집어넣습니다.

```
|                |---------------------------------------------------|
|  [홈]          |                                                   |
|  [내 몬스터]   |             (컨텐츠 영역)                         |

|  [상점]        |             또는 상점 아이템 목록                 |
|  [설정]        |             또는 랭킹 리스트                      |
|                |                                                   |
+----------------+---------------------------------------------------+
```
